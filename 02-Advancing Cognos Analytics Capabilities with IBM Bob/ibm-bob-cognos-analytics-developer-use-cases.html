2 · Data Module Engineering
🔍
Reverse-engineer any data module into human-readable docs
Read a module's full spec and metadata, generate schema documentation automatically
Works today get-module · get-module-metadata · list-modules

Just demonstrated live: Bob called get-module-metadata on Go Sales Data Modules and produced a complete table-by-table schema with primary keys, foreign keys, data types, usage classifications, and relationships — from raw JSON that was 430k characters long.

  1. Call get-module to get the module spec (useSpec, querySubjects)

  3. Parse all querySubjects, items, normalization groups into structured docs
  4. Output ERD-style relationships and column dictionaries
⚡ High impact — documentation that used to take days to write manually is generated in seconds.
Tools used: list-modules, get-module, get-module-metadata
✏️
AI-assisted data module creation from a database schema

Works today list-datasource-tables · get-datasource-schemas · create-module

"Create a data module for the HR_Workforce datasource covering the EMPLOYEE and DEPARTMENT tables." Bob lists the available schemas/tables, constructs the full module definition JSON following the Cognos spec (version, container, useSpec, querySubject), and calls create-module.

⚡ High impact — reduces hours of manual data modelling to minutes. Ensures correct JSON structure every time.
Tools used: list-datasources, list-datasource-connections, list-connection-signons, get-datasource-schemas, list-datasource-tables, create-module

Gaps — High-Value Tools Missing from Current MCP Server
1. Add Favorite — POST /bi/v1/users/~/favorites

Currently only GET is implemented. A single add-favorite tool registering POST /users/~/favorites with {"id": objectId} would close this. Tried this session — the endpoint exists but the tool is missing.

2. Create Datasource — POST /api/v1/datasources

Supports creating datasource + connection + signon in one atomic call. Critical for environment automation and CI/CD. The API supports it fully; just needs a new tool in index.ts.

3. Run Report (RDS) needs search-path fallback

The current run-report uses POST /disp/rds/reportData/report. The content-store based execution via /api/v1/execute would be more reliable. Worth adding an alternative execution path.

4. User / Role Management — POST /users, POST /roles/{id}/members

No write tools for user/role management exist yet. Adding create-user, assign-role, and remove-role would unlock full admin automation scenarios.

5. API Key management — POST /security/login_api_keys

Needed for CI/CD pipeline setup. The API key endpoint is documented and available; it just needs to be exposed as an MCP tool with appropriate security warnings.
